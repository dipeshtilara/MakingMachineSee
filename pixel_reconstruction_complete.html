Concept 01

What Exactly Is a Pixel?

A pixel (picture element) is the smallest unit of an image — a tiny coloured square storing three numbers: Red, Green, Blue, each from 0–255. Drag the sliders to mix any colour.

Colour mixing rules

R=255, G=0, B=0 → 🔴 Red R=0, G=255, B=0 → 🟢 Green R=0, G=0, B=255 → 🔵 Blue R=255, G=255, B=0 → 🟡 Yellow R=255, G=0, B=255 → 🟣 Magenta R=0, G=255, B=255 → 🩵 Cyan R=255, G=255, B=255 → ⬜ White R=0, G=0, B=0 → ⬛ Black

Memory cost

Each channel → 1 byte (0–255) 1 pixel → 3 bytes 1 MP image → ~3 MB (raw) 12 MP phone → ~36 MB (raw)

Key Takeaway

Every pixel stores exactly 3 integers — no colour names, only numbers.
The range is 0 (none) to 255 (maximum) for each of Red, Green, Blue.
Combining 256 × 256 × 256 = 16,777,216 possible colours from just three numbers.
Concept 02

Three Types of Digital Images

Not all images use RGB. Click each type to see how pixels are stored differently — and why each type is used in specific situations.

8×8 binary image — click to toggle

Values: 0 = ⬛ Black · 1 = ⬜ White

Binary Image

Each pixel is either 0 (black) or 1 (white) — just 1 bit per pixel. Used where colour is irrelevant and speed matters.

Pixel ∈ {0, 1} Bits per pixel: 1 Memory (8×8): 8 bytes Use cases: • OCR (document scanning) • Barcode / QR reading • Edge detection output • Fingerprint matching
1 bit per pixel Fastest to process Smallest file size
8×8 grayscale — hover to see brightness

Values: 0 = Black → 255 = White

Grayscale Image

Each pixel is a single brightness value from 0 (black) to 255 (white). No colour — just how bright each point is.

Pixel ∈ {0 ... 255} Bytes per pixel: 1 Array shape: [H, W] vs RGB: 3× smaller! Use cases: • Medical scans (X-ray, MRI) • Face detection (faster) • Satellite thermal imaging • Night-vision cameras
1 byte per pixel 3× smaller than RGB Shape: [H, W]
8×8 RGB — hover to inspect

Values: (R, G, B) per pixel

RGB Colour Image

Each pixel stores three values — Red, Green, Blue. The most common image format. Every photo, video frame, and website uses this.

Pixel = (R, G, B) Bytes per pixel: 3 Array shape: [H, W, 3] 16,777,216 colours possible Use cases: • All photographs • Video frames • Computer vision input • Display screens
3 bytes per pixel Shape: [H, W, 3] 16M+ colours

Comparison at a Glance

Binary: 1 bit/pixel → fastest, smallest. Grayscale: 1 byte/pixel → 3× smaller than RGB. RGB: 3 bytes/pixel → full colour.
Converting RGB to Grayscale: brightness = 0.299R + 0.587G + 0.114B (human eyes are most sensitive to green).
Exam tip: Grayscale array shape is [H, W]. RGB array shape is [H, W, 3]. Binary is [H, W] with values only 0 or 1.

Concept 04

Stored as a 3D Matrix

All pixel values live in memory as a 3D array shaped [Height × Width × 3]. Click the channel buttons to isolate R, G, or B. Hover cells to see coordinates.

5×5 sample matrix

Python / NumPy syntax

# Shape: [Height, Width, 3] img = np.array([ [[210,80,120],[200,75,115],...], [[195,70,110],[205,78,118],...], ... ]) img[0][0] → [210, 80, 120] img[0][0][0] → 210 ← Red img[0][0][1] → 80 ← Green img[0][0][2] → 120 ← Blue # Grayscale: shape [H, W] # RGB: shape [H, W, 3]

Why 3 dimensions?

Dim 1 = row (Y). Dim 2 = column (X). Dim 3 = channel (0=R, 1=G, 2=B). The same (row, col) address gives you one full pixel's colour when you read all 3 channels.

EXAM: shape (H,W,3) for colour · (H,W) for grayscale

Same data = same image, always

The matrix stores every pixel's exact colour with no approximation — pure numbers.
If you save and reload the same array, you always reconstruct the identical image.
Image files (PNG, BMP) are essentially these matrices, stored with optional compression on top.
Concept 05

RGB = Three Separate Layers

An RGB image is actually three independent grayscale images stacked. Each layer (channel) holds only the intensity of one colour. Click a channel to see it alone — this is how CNNs and filters work on images.

Click a channel to isolate it

Full RGB (combined)

All three channels merged — the colour image you normally see

Red channel only

How much red each pixel contributes. Brighter = more red.

Green channel only

Human eyes are most sensitive to green — it carries most detail.

Blue channel only

Least perceptually sensitive — skies, shadows, cool tones.

RGB → 3 grayscale images stacked Shape per channel: [H, W] Stack together: [H, W, 3] Filter on R only → img[:,:,0] Filter on G only → img[:,:,1] Filter on B only → img[:,:,2]
8×8 channel preview

Why this matters

CNNs (Convolutional Neural Networks) apply filters separately to each channel. Understanding channels helps you understand why a colour image has 3× more data than grayscale, and why CV models process them independently before combining.

Red channel = img[:,:,0] Green = img[:,:,1] Blue = img[:,:,2]

The reconstruction rule

Start at row 0, column 0. Read (R,G,B). Paint pixel at (0,0). Move right column by column.
When a row ends, drop to the next row. Repeat until all pixels are placed — that's the complete image.
Same pixel data → same image, always. Pixels are precise instructions, not random colours.
Advanced 01

Noise & Why Preprocessing Matters

Real cameras add random errors to pixel values — this is called noise. A pixel that should be (200, 150, 100) becomes (213, 138, 117) due to sensor heat or low light. Drag the slider to add noise and see the effect.

Clean Image

Pixel values exactly as stored

Noisy Image (±0)

Random errors added to each channel

0

Types of image noise

Gaussian noise: Random ± variations to all pixels Cause: low light, sensor heat Looks: overall graininess Salt & Pepper noise: Random pixels become 0 or 255 Cause: faulty sensors, data errors Looks: white/black speckles Compression artefacts: Blocky patterns in JPEG Cause: aggressive compression

How preprocessing fixes it

Problem → Solution Noisy pixels → Gaussian Blur (averages neighbours) → Median Filter (replaces with median) Wrong brightness → Normalisation (0–255 → 0.0–1.0) Wrong size → Resize to model input (e.g. 224×224) Too dark/bright → Histogram Equalisation
Noise = random errors Blur = noise remover

Why this connects to the CV Pipeline

Noise is why Step 2 of the CV pipeline (Preprocessing) exists — raw images are never clean enough to feed directly to a model.
A model trained on clean images will perform badly on noisy ones unless preprocessing standardises the input first.
Exam answer: "Preprocessing removes noise, resizes, and normalises images so the model receives consistent, clean input."
Advanced 02

How Image Compression Works

Raw images are huge. Compression shrinks file size. Click each quality level to see how the same image looks — and how many bytes it costs. There are two types: Lossless (no quality loss) and Lossy (smaller but degrades quality).

Select a quality level above

Click one of the four cards to see how compression changes the pixel data and file size.

Lossless vs Lossy

LOSSLESS (PNG, BMP, GIF): • Exact pixel values preserved • Larger files • Used: medical, legal, screenshots LOSSY (JPEG): • Some pixel data discarded • Much smaller files • Human eye can't see lost detail • Used: photos, web images Rule: quality ↓ → file size ↓ → pixelation ↑
PNG = Lossless JPEG = Lossy BMP = Uncompressed

Key Compression Facts

Raw image = every pixel's exact RGB values = large file. Compression reduces this by finding patterns.
JPEG reduces file size by averaging out fine colour differences the human eye can't easily perceive.
PNG is lossless — it compresses by finding repeating patterns but preserves every original pixel value exactly.
For CV/AI: always prefer lossless formats — lossy compression introduces artefacts that can confuse models.
Advanced 03

Resolution vs Image Size

Students confuse two different things. Image dimensions (width × height in pixels) and PPI (pixels per inch — print quality). Click each card to understand the difference interactively.

What changes on screen vs print

On a SCREEN: Only pixel count matters. More pixels = more detail shown. PPI setting doesn't affect display. On PRINT: PPI determines physical size. Same pixel count at: 72 PPI → large, rough print 300 PPI → small, sharp print Formula: Print width = pixel width ÷ PPI

Common image sizes

Thumbnail : 150 × 150 px Social media : 1080 × 1080 px Full HD : 1920 × 1080 px 4K Ultra HD : 3840 × 2160 px 12MP camera : 4000 × 3000 px Medical scan : 512 × 512 px (standard) CNN input : 224 × 224 px (ImageNet)
More pixels ≠ better always CNN needs fixed size input

Remember the distinction

Dimensions = how many pixels wide and tall the image is. This determines what the computer stores.
PPI = how many pixels fit in one inch when printed. This only affects physical print quality, not on-screen quality.
For CV/AI models: images are always resized to a fixed dimension (e.g. 224×224) before processing — PPI is irrelevant.
Advanced 04

Pixel Filters & Kernels

A filter (kernel) is a small 3×3 grid of numbers that slides over the image. At each pixel, it multiplies its neighbours by the kernel values and sums them to produce a new pixel. This is how edge detection, blurring, and sharpening work — and the foundation of CNNs.

Choose a filter type
3×3 kernel values

Blur

Averages each pixel with its 8 neighbours. All weights equal → smooth, soft result. Removes noise but loses sharpness.

new_pixel = (1/9) × sum of 9 neighbours = average of the 3×3 area Effect: smooths out sharp differences Used for: noise removal, preprocessing
Hover pixels to see kernel position

How it works step by step

1. Place 3×3 kernel over a pixel 2. Multiply each neighbour by the kernel value above it 3. Sum all 9 products 4. Result = new pixel value 5. Slide kernel one pixel right 6. Repeat for every pixel Hover the image grid above → to see the kernel sliding.

Why Filters Matter for CV

Edge detection filters find boundaries between objects — this is the first step in "feature extraction" in the CV pipeline.
Blur filters are used in preprocessing to remove noise before the image goes to a model.
CNNs (Convolutional Neural Networks) learn their own filters automatically from training data — they are not hand-coded like these.
Exam key: "Convolution = sliding a kernel over every pixel and computing a weighted sum of its neighbourhood."
